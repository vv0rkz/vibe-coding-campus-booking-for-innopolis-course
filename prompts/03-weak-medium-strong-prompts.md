# Промпт 3 — Сравнение слабого, среднего и сильного промпта

## Контекст
Урок 2. Задача: сравнить качество ответа ИИ при разном уровне промптов на одной задаче.

## Задача для сравнения
Описать целевую аудиторию сервиса бронирования ресурсов кампуса.

---

## Слабый промпт

> Опиши целевую аудиторию сервиса бронирования.

### Результат
Общий, поверхностный ответ: «целевая аудитория — студенты и преподаватели, которым нужно бронировать помещения». Без деталей, без ролей, без контекста использования. Бесполезно для продуктовой работы.

---

## Средний промпт

> Опиши целевую аудиторию веб-сервиса бронирования ресурсов университетского кампуса. Укажи роли пользователей и основные сценарии использования для каждой роли.

### Результат
Выделены 3 роли (студент, преподаватель, админ), для каждой — 2–3 сценария. Уже полезнее, но нет портрета пользователя, нет болей, нет JTBD.

---

## Сильный промпт

> Ты — продуктовый аналитик с опытом в EdTech. Я разрабатываю MVP сервиса бронирования ресурсов университетского кампуса (переговорные, коворкинги, оборудование, консультации).
>
> Опиши целевую аудиторию продукта. Для каждого сегмента укажи:
> - Роль и портрет (возраст, контекст, привычки)
> - Текущий способ решения задачи (as-is) и его проблемы
> - JTBD — какую «работу» продукт выполняет для этого пользователя
> - Частота использования и ключевой сценарий
>
> Представь в виде таблицы. Проверь свой результат на полноту.

### Результат
Детальная таблица с 3 сегментами: портрет, as-is процесс, боли, JTBD, частота, ключевой сценарий. Ответ можно напрямую использовать в продуктовой документации.

---

## Вывод

| Критерий | Слабый | Средний | Сильный |
|----------|--------|---------|---------|
| Роль для модели | нет | нет | да |
| Контекст проекта | нет | частично | да |
| Формат ответа | нет | нет | да (таблица) |
| Глубина ответа | поверхностно | нормально | детально |
| Практическая польза | низкая | средняя | высокая |

Чем точнее роль, контекст, задача и формат — тем полезнее ответ ИИ.
