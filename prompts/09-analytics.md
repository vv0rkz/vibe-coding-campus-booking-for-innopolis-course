# Промпты — hw9: аналитика (Microsoft Clarity)

## Контекст

CampusBook — статический фронт + Supabase, задеплоен на Vercel. Нужно прикрутить Clarity, но без хардкода ID в html и с возможностью выключить аналитику на локалке.

---

## Промпт 1 — интеграция без хардкода

> В моём проекте одна страница `index.html` и один `script.js`. Хочу подключить Microsoft Clarity, но без того чтобы хардкодить ID прямо в html.
> Сделай так:
> - ID живёт в `supabase-config.js` как `window.CAMPUSBOOK_CLARITY_ID`, пусто = отключено,
> - в `script.js` функция `initClarity()`, которая инжектит стандартный snippet Clarity через DOM если ID задан,
> - вызывается из `DOMContentLoaded` до Supabase init,
> - нигде не импортируется внешний CDN, если ID не задан (нулевой оверхед в dev-режиме).

## Промпт 2 — события для воронки

> На сайте есть секции: лендинг, каталог ресурсов, календарь (booking), мои брони, auth-модалка, paywall.
> Ключевые действия: пользователь открыл календарь, выбрал слот, подтвердил бронь, упёрся в paywall.
> Предложи минимальный набор кастомных событий Clarity (`clarity('set', ...)` и `clarity('event', ...)`), чтобы построить воронку «зашёл → дошёл до календаря → выбрал слот → забронировал».

## Промпт 3 — приватность

> Подсай политику приватности Clarity: что по умолчанию маскируется, что нет, как настроить strict masking для полей с email и паролями, как дать opt-out пользователю. В конце — один абзац для пункта «Cookies» в политике сайта.

## Промпт 4 — проверка после деплоя

> Сайт на Vercel, Clarity должен прислать первую запись. Как быстро проверить что трекер работает: что смотреть в DevTools Network, за какое время появляются первые Recordings, как отличить «не работает» от «работает, но трафика мало».
