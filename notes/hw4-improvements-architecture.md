# Домашнее задание 4 — Улучшения MVP и архитектура

## Проект: CampusBook — сервис бронирования ресурсов кампуса

---

## 1. Улучшения MVP (3 штуки)

### Улучшение 1: Поиск по ресурсам

**Что:** Текстовое поле поиска в секции каталога. Работает в реальном времени — при вводе текста карточки фильтруются по названию и описанию.

**Зачем:** Когда ресурсов станет больше (10+), прокручивать весь каталог неудобно. Быстрый поиск экономит время пользователя.

**Как реализовано:** В `script.js` добавлен обработчик `input` на поле поиска, который фильтрует массив `RESOURCES` по вхождению подстроки в `name` и `description` (регистронезависимо) и перерисовывает карточки.

### Улучшение 2: Статистика бронирований

**Что:** Блок над списком бронирований, показывающий: общее количество бронирований, количество активных, количество отменённых.

**Зачем:** Пользователь сразу видит общую картину без необходимости считать вручную. Для администратора это первый шаг к аналитике загрузки ресурсов.

**Как реализовано:** Функция `renderStats()` считает количество бронирований по статусу из localStorage и отображает 3 счётчика в компактных карточках над списком.

### Улучшение 3: Экспорт бронирований

**Что:** Кнопка «Скачать JSON» в секции бронирований, которая скачивает все данные из localStorage в файл `campusbook_bookings.json`.

**Зачем:** Позволяет сохранить резервную копию данных, перенести их на другое устройство или использовать для анализа. В перспективе — импорт в базу данных при миграции с localStorage на Supabase.

**Как реализовано:** Функция `exportBookings()` создаёт Blob из JSON-данных localStorage и инициирует скачивание через временную ссылку.

## 2. Таблица архитектуры MVP

| Слой | Компонент | Описание | Технология |
|------|-----------|----------|------------|
| **Frontend** | Header + навигация | Sticky-шапка с якорными ссылками | HTML, CSS |
| | Hero-секция | Заголовок, описание, CTA | HTML, CSS |
| | Каталог ресурсов | Карточки с фото, фильтрация по типу, поиск | HTML, CSS, JS |
| | Форма бронирования | Ввод данных, выбор ресурса/даты/времени, валидация | HTML, JS |
| | Список бронирований | Отображение, отмена, статистика, экспорт | HTML, JS |
| **Backend (логика)** | Валидация формы | Проверка корректности данных (время, обязательные поля) | JS |
| | Проверка конфликтов | Поиск пересечений по ресурсу + дате + времени | JS |
| | Управление бронированиями | Создание, отмена, фильтрация, сортировка | JS |
| | Экспорт данных | Сериализация данных в JSON, скачивание файла | JS |
| **Database (данные)** | Ресурсы кампуса | Статический массив объектов (id, name, type, floor, capacity) | JS (const) |
| | Бронирования | Массив объектов (id, resourceId, userName, date, time, status) | localStorage |

```

    │
    ▼
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  Frontend   │────▶│  Backend (JS)    │────▶│  Database        │
│             │     │                  │     │  (localStorage)  │
│ - Каталог   │     │ - Валидация      │     │                 │
│ - Форма     │◀────│ - Конфликт-чек   │◀────│ - bookings[]    │

│ - Статы     │     │ - Экспорт JSON   │     │                 │
└─────────────┘     └──────────────────┘     └─────────────────┘
```
