# Домашнее задание 3 — Лендинг с интерактивными сценариями

## Проект: CampusBook — сервис бронирования ресурсов кампуса

---

## Что сделано

Создан одностраничный лендинг (`projects/campus-booking/`) на чистом HTML + CSS + JS с двумя интерактивными сценариями и сохранением данных в localStorage.

### Структура файлов

```
projects/campus-booking/
├── index.html   — разметка страницы
├── style.css    — стили (адаптивная верстка, современный UI)
└── script.js    — логика: каталог, фильтры, бронирование, localStorage
```

---

## Интерактивные сценарии

### Сценарий 1: Бронирование ресурса

1. Пользователь заполняет форму: имя, ресурс, дата, время начала/окончания, цель.
2. При отправке система проверяет:
   - Время начала < время окончания
   - Нет конфликта с существующими бронированиями на тот же ресурс и дату
3. Если проверки пройдены — бронь сохраняется в `localStorage`, появляется уведомление об успехе.
4. Если конфликт — показывается ошибка с деталями (кто и когда уже занял).

### Сценарий 2: Управление бронированиями

1. Секция «Мои бронирования» отображает все брони из `localStorage`, отсортированные по дате создания.
2. Активные брони можно **отменить** — кнопка «Отменить» меняет статус на `cancelled`.
3. Отменённые брони отображаются приглушённо со статусом «Отменено».
4. Данные сохраняются между перезагрузками страницы (localStorage).

---

## Дополнительные функции

- **Каталог ресурсов** — карточки с фото, описанием, этажом и вместимостью.
- **Фильтрация по типу** — кнопки «Переговорные / Коворкинги / Оборудование / Консультации» динамически фильтруют карточки.
- **Валидация формы** — HTML5 required + JS-проверка пересечения временных слотов.
- **Адаптивная верстка** — корректно отображается на мобильных устройствах.

---

## Проверка localStorage

1. Открыть `index.html` в браузере
2. Создать бронирование через форму
3. Открыть DevTools → Application → Local Storage
4. Ключ `campusbook_bookings` содержит массив объектов бронирований
5. После перезагрузки страницы бронирования сохраняются
