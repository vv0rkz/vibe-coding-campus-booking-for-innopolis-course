# Домашнее задание 8 — Публикация MVP

## Проект: CampusBook — сервис бронирования ресурсов кампуса

---

## Цель

Опубликовать MVP так, чтобы на него можно было зайти по постоянной ссылке, и понять, какие варианты деплоя существуют и почему выбран именно этот.

---

## Ключевые понятия

| Термин | Что это | В нашем проекте |
|---|---|---|
| **Деплой** | Процесс публикации проекта в интернет | `vercel --prod` из `projects/campus-booking/` |
| **Хостинг** | Место, где живут файлы сайта | Vercel CDN |
| **Домен** | Человеко-читаемый адрес сайта | `campus-booking.vercel.app` (subdomain Vercel, бесплатный) |
| **Сервер** | Компьютер, который отдаёт страницы 24/7 | Edge-сеть Vercel — статика раздаётся с ближайшего edge-ноды; база `bookings` и Auth живут на серверах Supabase |

---

## Виды публикаций

| Тип | Что можно | Примеры | Подходит для |
|---|---|---|---|
| **Статическая** | Только HTML/CSS/JS, нет своего бэкенда | Лендинг, портфолио | Лендинги, демо |
| **Динамическая** | Есть база, auth, API | SaaS-продукт | Реальный MVP с логикой |

CampusBook формально **статический фронтенд**, но у него есть «динамическая обвязка» — Supabase (PostgreSQL + Auth + REST API через PostgREST). Это упрощает жизнь: статику можно раздавать из любого CDN, а всю серверную часть берёт на себя Supabase. Отдельный backend-сервер не нужен.

---

## Варианты деплоя

| Вариант | Плюсы | Минусы | Цена |
|---|---|---|---|
| **htmlpreview.github.io** | 0 настроек, работает из git-тега | Нет env, блокируются CDN-скрипты адблокерами, нет кастомного домена | Free |
| **GitHub Pages** | Встроен в GitHub, домен `*.github.io` | Только статика, нужен правильный `base` в путях, CI-сборка через Actions | Free |
| **Vercel** | CLI `vercel --prod`, мгновенный preview на каждый PR, нормальные env-переменные | Привязка к вендору | Free на hobby-тариф |
| **Netlify** | Аналогично Vercel, простой UI | — | Free |
| **VPS** (любой хостер) | Полный контроль, любой стек | Ручная настройка nginx, сертификатов, автозапуска; нужны DevOps-навыки | 200–500 ₽/мес |

### Ограничения бесплатных вариантов (лекция)

- лимиты запросов (на Supabase free — 500 MB storage, 50k MAU, 2 GB bandwidth),
- «засыпает» сервер — относится к бесплатным динамическим хостингам (Render/Railway), к Vercel-статике **не** относится,
- нестабильность пиков, нет SLA.

### Когда нужен платный сервер

- реальный трафик с конверсией,
- собственный backend с тяжёлыми задачами (ML, очереди, cron),
- стабильная латентность / SLA,
- критичная БД, не помещающаяся во free-лимиты Supabase.

---

## Что выбрано для CampusBook и почему

**Vercel (статика) + Supabase (база + auth).**

Почему:
1. Фронт — vanilla HTML/CSS/JS, сборка не нужна → идеален для Vercel/Netlify из коробки.
2. Supabase-anon-key можно вставлять в клиент (защита через RLS). Service-role-ключ в клиент не идёт.
3. Vercel CLI `vercel --prod` разворачивает за 10 секунд, выдаёт HTTPS-сертификат автоматически.
4. Нулевая цена на учебный MVP.

**Живой сайт:** [https://campus-booking.vercel.app](https://campus-booking.vercel.app)

## Шаги деплоя

```bash
# 1. Установить Vercel CLI (один раз)
npm i -g vercel

# 2. Залогиниться
vercel login

# 3. Из папки проекта
cd projects/campus-booking

# 4. Первый запуск — выбрать scope, link project
vercel

# 5. Продакшн-деплой
vercel --prod
```

### Переменные окружения

Для статического фронта CampusBook env не нужны — ключи Supabase прямо в `supabase-config.js` (это **anon key**, безопасный при правильных RLS). При расширении до серверных функций (webhook платёжки из hw11) переменные ставятся в **Vercel Dashboard → Settings → Environment Variables**, а клиентский код туда не должен получать доступ к `SERVICE_ROLE_KEY`.

### Supabase Auth после деплоя

В **Supabase → Authentication → URL Configuration**:
- **Site URL:** `https://campus-booking.vercel.app`
- **Redirect URLs:**
  - `https://campus-booking.vercel.app/*`
  - `http://localhost:8000/*` (для локальной разработки)

Иначе магические ссылки и password reset будут редиректить не туда.

## Как проверить, что всё работает

1. Открыть [campus-booking.vercel.app](https://campus-booking.vercel.app) в инкогнито — загрузка без ошибок в консоли.
2. Зарегистрироваться новым email — получить письмо подтверждения (если включено).
3. Войти, создать бронь — проверить в Supabase Table Editor, что строка в `bookings` появилась и `user_id` совпадает.
4. В DevTools → Network должны быть только `supabase.co` и домены ассетов (fonts, Unsplash для картинок). Никаких 404 на локальные пути.
5. Lighthouse (DevTools → Lighthouse) на деплое должен дать performance > 90 (страница статическая, картинки lazy).
