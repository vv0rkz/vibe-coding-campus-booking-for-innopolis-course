# Домашнее задание 2 — Пользователь, проблема, функции и клиентский путь

## Проект: CampusBook — сервис бронирования ресурсов кампуса

---

## 1. Пользователь и его контекст

### Основной пользователь: студент университета

**Портрет:** Студент 18–25 лет, активно участвует в командных проектах, хакатонах и учебных группах. Технически грамотен, привык решать задачи через приложения и мессенджеры. Ценит скорость и прозрачность.

**Повседневная ситуация:** Студенту нужно обсудить проект с командой из 4 человек. Ему нужна переговорная комната на 1,5 часа сегодня после 15:00. Сейчас он пишет в общий чат: «Кто-нибудь знает, свободна ли переговорка на 3-м этаже?» — ждёт ответа, потом выясняет, что она занята, ищет другую, снова спрашивает... В итоге тратит 15–20 минут на организацию, которая должна занимать секунды.

### Вторичные пользователи

- **Преподаватель/ментор** — публикует слоты для консультаций, видит записавшихся студентов.
- **Администратор кампуса** — добавляет ресурсы, модерирует заявки, отслеживает загрузку.

## 2. Проблема (JTBD)

> **Когда** я хочу забронировать переговорку или оборудование для командной работы,
> **я хочу** быстро увидеть, что свободно, и забронировать за пару кликов,
> **чтобы** не тратить время на переписку в чатах и не столкнуться с конфликтом бронирования.

### Текущие боли

| Боль | Последствие |
|------|-------------|
| Нет единого места для просмотра свободных ресурсов | Студенты не знают, что доступно, и тратят время на выяснение |
| Бронирование через чаты и устные договорённости | Конфликты: два человека приходят в одно время на одно место |
| Нет отмены / переноса | «Призрачные» бронирования блокируют ресурсы, которые никто не использует |
| Нет уведомлений | Студент забывает про бронь или не знает об изменениях |
| Нет статистики для администрации | Невозможно оценить загрузку ресурсов и планировать |

## 3. Ключевые функции MVP (5 функций)

### Ф1. Каталог ресурсов кампуса

Список всех доступных для бронирования ресурсов: переговорные комнаты, учебные аудитории, коворкинг-зоны, оборудование (проекторы, доски). Каждый ресурс — карточка с фото, описанием, вместимостью, локацией и доступным расписанием. Фильтрация по типу, этажу, вместимости.

### Ф2. Календарь доступности и бронирование

Интерактивный календарь с таймслотами для каждого ресурса. Студент видит свободные и занятые слоты, выбирает нужный и бронирует в один клик. Система не позволяет забронировать уже занятый слот — конфликты исключены на уровне логики.

### Ф3. Личный кабинет пользователя

Авторизация через email. В кабинете — список активных бронирований, история прошлых, возможность отменить или перенести бронь. Роль пользователя (студент / преподаватель / админ) определяет доступные действия.

### Ф4. Уведомления

Подтверждение бронирования, напоминание за 30 минут до начала, уведомление об отмене. На первом этапе — через email, в перспективе — push и Telegram-бот.

### Ф5. Админ-панель

Администратор добавляет и редактирует ресурсы, видит все бронирования, может модерировать заявки (одобрять/отклонять для определённых ресурсов), просматривает базовую статистику загрузки.

## 4. Клиентский путь (User Flow)

### Сценарий: студент бронирует переговорку

```
┌─────────────────────────────────────────────────────┐
│  1. ВХОД                                            │
│  Студент открывает CampusBook → логинится по email  │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│  2. ПОИСК РЕСУРСА                                   │
│  Видит каталог → фильтрует: "переговорки, 3 этаж,  │
│  на 4+ человека" → выбирает подходящую комнату      │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│  3. ВЫБОР ВРЕМЕНИ                                   │
│  Открывает календарь ресурса → видит свободные      │
│  слоты на сегодня → выбирает 15:00–16:30            │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│  4. ПОДТВЕРЖДЕНИЕ БРОНИ                             │
│  Указывает цель (проектная встреча) → нажимает      │
│  "Забронировать" → получает подтверждение на экране │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│  5. УВЕДОМЛЕНИЕ                                     │
│  Получает email с деталями брони.                   │
│  За 30 мин до встречи — напоминание.                │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│  6. ИСПОЛЬЗОВАНИЕ / УПРАВЛЕНИЕ                      │
│  Приходит на встречу. Если планы изменились —       │
│  отменяет бронь в личном кабинете, слот             │
│  освобождается для других.                          │
└─────────────────────────────────────────────────────┘
```

### Сценарий: преподаватель открывает слоты для консультаций

1. Преподаватель логинится → переходит в «Мои ресурсы»
2. Создаёт слоты для консультаций (дата, время, кабинет)
3. Студенты видят свободные слоты в каталоге → записываются
4. Преподаватель видит список записавшихся в личном кабинете

### Сценарий: администратор управляет ресурсами

1. Админ логинится → открывает админ-панель
2. Добавляет новый ресурс (название, фото, вместимость, расписание)
3. Просматривает все бронирования за неделю
4. Видит статистику: самые популярные ресурсы, пиковые часы, % загрузки

## 5. MVP Scope — что входит, а что нет

### В MVP входит (v1.0)

- Каталог ресурсов с карточками и фильтрами
- Календарь доступности для каждого ресурса
- Бронирование с защитой от конфликтов
- Авторизация по email
- Личный кабинет (мои брони, отмена)
- Базовая админ-панель (CRUD ресурсов, просмотр броней)

### Не входит в MVP (backlog)

- Telegram-бот для бронирования
- Push-уведомления
- Интеграция с университетской SSO/LDAP
- Рейтинг и отзывы на ресурсы
- Повторяющиеся бронирования (recurring)
- Мобильное приложение
- QR-коды на дверях для быстрого бронирования
