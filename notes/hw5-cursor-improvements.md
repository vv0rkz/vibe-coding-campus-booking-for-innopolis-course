# Домашнее задание 6 — Улучшения MVP через Cursor

## Проект: CampusBook — сервис бронирования ресурсов кампуса

---

## Что сделано

Используя Cursor с встроенным AI, внесены 3 улучшения в MVP. Все изменения — через промпты в Cursor, без ручного написания кода.

---

## Улучшение 1: Тёмная тема (Dark Mode)

**Что:** Переключатель тёмной/светлой темы в шапке сайта. При нажатии меняются цвета фона, текста, карточек и форм.

**Зачем:** Студенты часто работают вечером и ночью — тёмная тема снижает нагрузку на глаза. Это стандартная функция современных веб-приложений, которую ожидают пользователи.

**Как реализовано через Cursor:**
- Промпт: *«Добавь переключатель тёмной темы в header. При нажатии должен меняться класс на body. Создай CSS-переменные для цветов и два набора значений: для светлой и тёмной темы. Сохраняй выбор темы в localStorage.»*
- Cursor добавил кнопку-тогл в `index.html`, CSS-переменные в `style.css` и JS-логику сохранения темы в `script.js`

---

## Улучшение 2: Подтверждение перед отменой бронирования

**Что:** При нажатии кнопки «Отменить» появляется модальное окно с подтверждением: «Вы уверены, что хотите отменить бронирование?»

**Зачем:** Защита от случайных нажатий. Отмена — деструктивное действие, которое нельзя откатить. Без подтверждения пользователь может потерять бронь одним случайным кликом.

**Как реализовано через Cursor:**
- Промпт: *«Добавь модальное окно подтверждения перед отменой бронирования. Модалка должна показывать название ресурса, дату и время. Кнопки: Отменить бронь (красная) и Назад (серая). Стили модалки — в style.css.»*
- Cursor создал HTML-модалку, обновил `cancelBooking()` для показа диалога и добавил стили оверлея

---

## Улучшение 3: Сортировка бронирований

**Что:** Выпадающий список над секцией «Мои бронирования» для сортировки: по дате создания (новые первые), по дате бронирования, по статусу (сначала активные).

**Зачем:** Когда бронирований накапливается много, пользователю нужно быстро найти нужное. Сортировка по дате бронирования помогает видеть ближайшие встречи, а сортировка по статусу — отделить активные от отменённых.

**Как реализовано через Cursor:**
- Промпт: *«Добавь select для сортировки бронирований. Три варианта: по дате создания (по умолчанию), по дате бронирования, по статусу. Обнови renderBookings() для поддержки сортировки. Сохраняй выбранный вариант.»*
- Cursor добавил `<select>` в HTML, обновил логику сортировки в `renderBookings()` и подключил обработчик `change`

## Процесс работы с Cursor

### Шаг 1: Открыл проект в Cursor
Клонировал репозиторий с GitHub, открыл папку `projects/campus-booking/` в Cursor.

### Шаг 2: Изучил структуру через AI
Промпт: *«Объясни, что это за проект, из каких файлов он состоит простыми словами»*
Cursor описал: `index.html` — разметка, `style.css` — стили, `script.js` — логика и данные.

### Шаг 3: Добавил комментарии
Промпт: *«Добавь комментарии ко всем важным частям кода, чтобы было понятно, что и где»*
Cursor разметил основные функции и блоки.

### Шаг 4: Вносил улучшения
Каждое улучшение — отдельный промпт в Cursor. После каждого изменения проверял в браузере, что всё работает.

### Шаг 5: Закоммитил в GitHub
Все изменения запушены в основной репозиторий.

## Выводы

- Cursor позволяет вносить осмысленные изменения в код, даже не понимая каждую строку
- Важно формулировать промпт конкретно: не «сделай лучше», а «добавь select с тремя вариантами сортировки»
- После каждого изменения нужно проверять результат в браузере — AI иногда генерирует код, который выглядит правильно, но не работает
- Git помогает откатиться, если изменение сломало что-то
